What is useEffect?

The useEffect hook allows you to perform side effects in functional components. Side effects can include things like:

  • Fetching data from an API.
  • Subscribing to events.
  • Directly manipulating the DOM.

Hereâ€™s a simple example of how useEffect works:

import React, { useState, useEffect } from 'react';

function ExampleComponent() {
  const [count, setCount] = useState(0);

  useEffect(() => {
    console.log(`You clicked ${count} times`);
  }, [count]); // Only re-run the effect if count changes

  return (
    <div>
      <p>You clicked {count} times</p>
      <button onClick={() => setCount(count + 1)}>Click me</button>
    </div>
  );
}

export default ExampleComponent;

Note: The second argument of useEffect is the dependency array. It ensures the effect runs only when specified values change.

Common use cases

1. Fetching data

One of the most common uses of useEffect is fetching data from an API:

useEffect(() => {
  async function fetchData() {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    console.log(data);
  }
  fetchData();
}, []); // Empty dependency array means this runs only once after the component mounts

2. Subscribing to events

You can also use useEffect to subscribe to events like resizing the window:

useEffect(() => {
  function handleResize() {
    console.log('Window resized:', window.innerWidth);
  }

  window.addEventListener('resize', handleResize);

  return () => {
    window.removeEventListener('resize', handleResize); // Cleanup
  };
}, []);

Best practices for using useEffect

Here are some tips to make the most out of useEffect:

  1. Always clean up side effects
    When using effects like event listeners or timers, always return a cleanup function:

    useEffect(() => {
  const timer = setInterval(() => console.log('Tick'), 1000);
  return () => clearInterval(timer); // Cleanup
}, []);

  2. Avoid overusing effects
    Keep your effects focused on specific tasks. If you find yourself adding too much logic into one effect, consider splitting it into multiple effects.

  3. Use dependencies wisely
    Always declare all variables used inside your effect in the dependency array to avoid bugs.

Common pitfalls

Forgetting dependencies

If you forget to include dependencies in the array, your effect might not behave as expected:

// This will not re-run when 'props.value' changes!
useEffect(() => {
  console.log(props.value);
}, []);

Infinite loops

Be cautious of infinite loops caused by improperly updating state inside an effect:

// This will cause an infinite loop!
useEffect(() => {
  setState(state + 1); // Updating state triggers re-render
}, [state]);
